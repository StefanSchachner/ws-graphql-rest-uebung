== Projektbeschreibung

Eine Quarkus-Anwendung, die eine Todo-Liste über vier verschiedene Protokolle/Technologien bereitstellt:

* REST
* GraphQL
* WebSocket
* MQTT

Als Datenbank wird H2 verwendet (TCP im Dev-Modus, In-Memory im Prod-Modus).

== Technologien

* Java 25
* Quarkus 3.35.2
* H2 Datenbank
* Hibernate ORM Panache
* Eclipse Paho MQTT Client

== Projektstruktur

[source]
----
src/main/java/at/htl/
├── control/
│   └── TodoRepository.java       # Panache Repository
├── entity/
│   └── Todo.java                 # JPA Entity (id, todo, completed, userId)
├── resource/
│   ├── HelloGraphQLResource.java # GraphQL sayHello
│   ├── TodoGraphQLResource.java  # GraphQL allTodos
│   ├── TodoRestResource.java     # REST GET /todos
│   ├── TodoWebSocket.java        # WebSocket /todos-ws
│   └── TodoMqttClient.java       # MQTT Publisher
k8s/
└── deployment.yaml               # Kubernetes Deployment + Service + Ingress
----

== Endpoints

=== REST

[cols="1,1,3"]
|===
|Methode |Pfad |Beschreibung

|GET
|`/todos`
|Gibt alle Todos als JSON zurück
|===

*Beispiel-Response:*
[source,json]
----
[
  { "id": 1, "todo": "Do something nice for someone you care about", "completed": false, "userId": 152 },
  { "id": 2, "todo": "Memorize a poem", "completed": true, "userId": 13 }
]
----

=== GraphQL

Erreichbar unter `/graphql` bzw. GraphQL UI unter `/q/graphql-ui`.

*Query — alle Todos:*
[source,graphql]
----
query {
  allTodos {
    id
    todo
    completed
    userId
  }
}
----

*Query — Hello:*
[source,graphql]
----
query {
  sayHello(name: "HTL")
}
----

=== WebSocket

Verbindung auf `/todos-ws`. Beim Verbinden werden alle Todos als JSON-Array gesendet.

* Lokal: `ws://localhost:8080/todos-ws`
* LeoCloud: `wss://it210200.cloud.htl-leonding.ac.at/todos-ws`

*Testen im Browser (DevTools Console):*
[source,javascript]
----
const ws = new WebSocket("wss://it210200.cloud.htl-leonding.ac.at/todos-ws");
ws.onopen = () => console.log("verbunden");
ws.onmessage = (e) => console.log(JSON.parse(e.data));
----

=== MQTT

Beim Start der Anwendung werden alle Todos als JSON auf Topic `todos/update` gepublished.

* Broker: `broker.hivemq.com:1883`
* Topic: `todos/update`

*Testen mit HiveMQ Web Client:* https://www.hivemq.com/demos/websocket-client/[HiveMQ Web Client]

== Deployment

=== Voraussetzungen

* Docker
* kubectl
* minikube
* leocloud CLI
* GitHub Account (für ghcr.io)

=== Docker Image bauen

[source,bash]
----
./mvnw package -DskipTests
docker buildx build --platform linux/amd64 \
  -f src/main/docker/Dockerfile.jvm \
  -t ghcr.io/stefanschachner/ws-graphql-rest-uebung:latest \
  --push .
----

=== Minikube

[source,bash]
----
minikube start
kubectl config use-context minikube
kubectl apply -f k8s/deployment.yaml
----

=== LeoCloud

[source,bash]
----
leocloud auth login
kubectl config use-context leocloud
kubectl apply -f k8s/deployment.yaml -n student-it210200
----

Die Anwendung ist erreichbar unter: https://it210200.cloud.htl-leonding.ac.at
